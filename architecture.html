Project architecture

The project moved away from a single oversized class into a cleaner modular structure, where the UI, data layer, crypto layer, and services each have a more focused role.

main.py

Minimal entry point that starts the main Tkinter application window.

config.py

Constants, bundled resource paths, wallpaper folder, icon path, and shared configuration values.

db.py

SQLite access, schema setup, migrations, and transactional updates.

crypto.py

Password encryption and decryption, key derivation, and legacy compatibility support.

auth.py

Registration, login, session state, failed-attempt handling, and settings updates.

vault.py

Business logic around entries: saving, deleting, validating, decrypting, and migration of stored records.

ui/app.py

The visible application layer: themes, forms, list rendering, wallpapers, music, settings, and event handling.
